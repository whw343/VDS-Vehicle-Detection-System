## 二、YOLOv5 车辆检测

### 2.1 模型选择

选用 `yolov5su.pt`（YOLOv5 Small Ultralytics版，18.6MB），兼顾速度与精度。

### 2.2 车辆类别过滤

YOLOv5预训练80类COCO标签中，仅保留车辆相关4类：

| COCO ID | 类别 | 中文名 |
|---------|------|--------|
| 2 | car | 轿车 |
| 3 | motorcycle | 摩托车 |
| 5 | bus | 客车 |
| 7 | truck | 卡车 |

### 2.3 实现要点

```python
from ultralytics import YOLO

class VehicleDetector:
    VEHICLE_CLASSES = [2, 3, 5, 7]
    
    def load_model(self):
        self.model = YOLO('weights/yolov5su.pt')
    
    def detect(self, image_path):
        results = self.model(image_path, conf=0.45)
        # 过滤非车辆类别 → 返回bbox + crop
```

## 三、ResNet18 分类模型

### 3.1 架构设计

三个分类器共享相同的 ResNet18 骨干网络，仅最后全连接层不同：

```
ResNet18 Backbone (pretrained=False)
    ↓
Linear(512, 256) + ReLU + Dropout(0.3)
    ↓
Linear(256, N)  ← N=类别数
```

### 3.2 颜色分类模型

| 属性 | 值 |
|------|-----|
| 类别 | black, blue, brown, green, red, silver, white, yellow (8类) |
| 训练数据 | 任务8数据 (524张真实车辆图片) |
| 训练轮数 | 15 |
| 最佳验证准确率 | **94.29%** |
| 真实图测试(裁剪后) | 蓝色100%, 红色99% |

### 3.3 车型分类模型

| 属性 | 值 |
|------|-----|
| 类别 | bus, car, mini, truck (4类) |
| 训练数据 | 任务4数据 (285张真实车辆图片) |
| 训练轮数 | 20 |
| 最佳验证准确率 | **91.23%** |

### 3.4 品牌分类模型

| 属性 | 值 |
|------|-----|
| 类别 | 47种品牌（一汽卡车, 东风, 丰田, 五菱...） |
| 训练数据 | 阶段三-任务4数据 (3700张XML Pascal VOC标注) |
| 训练轮数 | 10 |
| 数据特点 | 使用XML bndbox裁剪车辆区域后训练 |

品牌列表从 `weights/brand_labels.txt` 动态加载，支持灵活扩展。

## 四、模型加载兼容性

三个分类器的权重保存格式为纯ResNet18 state_dict（无backbone前缀），加载时自动添加前缀：

```python
state_dict = torch.load(path, weights_only=True)
if not any(k.startswith('backbone.') for k in state_dict):
    state_dict = {'backbone.' + k: v for k, v in state_dict.items()}
```

## 五、统一推理管线

`inference.py` 提供端到端推理接口，串联所有模块：

```
图片输入 → YOLOv5检测 → 裁剪车辆区域
    ↓
车牌OCR → 颜色分类 → 车型分类 → 品牌分类
    ↓
compare.py 多因素比对 → vehicle-database.csv
    ↓
判定结果 (normal/suspicious) + 评分(0-100)
```

## 七、已知限制与改进方向

| 限制 | 影响 | 改进方向 |
|------|------|---------|
| OCR全景图识别率低(25%) | 车牌比对 | 增加车牌检测模型(HyperLPR) |
| 品牌47类准确率待提升 | 品牌比对 | 更多训练数据+更长训练 |
| 颜色模型域偏移 | 未裁剪图误差大 | 数据增强+域适应 |
| 无YOLOv5微调 | 车辆检测漏检 | 用3700张XML标注微调 |
