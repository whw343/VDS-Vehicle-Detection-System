## 一、项目需求

### 1.1 业务背景

随着汽车保有量爆发式增长，套牌车（克隆车牌车）现象屡禁不止。本项目构建基于AI的套牌车稽查系统，自动分析交通卡口图片，提取车辆结构化特征并与登记库比对，快速识别套牌嫌疑。

### 1.2 功能需求

| 编号 | 功能 | 描述 |
|------|------|------|
| F1 | 车辆检测 | 从卡口图片中定位车辆区域 |
| F2 | 车牌识别 | OCR提取车牌号（如鲁B62B23） |
| F3 | 颜色识别 | 分类车身颜色（黑/蓝/红/白等8色） |
| F4 | 车型识别 | 分类车辆类型（轿车/客车/卡车/微型车） |
| F5 | 品牌识别 | 分类车辆品牌（47种） |
| F6 | 套牌判定 | 多特征比对数据库，输出判定结果 |
| F7 | Web展示 | 上传图片→分析→结果展示 |

### 1.3 非功能需求

- 单张图片分析时间 < 5秒
- 车辆检测率 > 90%
- Web界面响应式设计
- 支持 JPG/PNG/BMP 格式

### 2.3 数据库设计

```
vehicle-database.csv
├── plateNo: 车牌号 (主键, 如"鲁B62B23")
└── carBrand: 车辆品牌 (如"江淮卡车")
总记录: 3581条 (原始6736条, 清洗去重3154条)
品牌种类: 49种
```

### 2.4 套牌判定规则

```
IF 车牌不存在于数据库 → 套牌嫌疑 (评分=10)
IF 车牌匹配但品牌不符 → 套牌嫌疑 (评分=40)
IF 车牌+品牌均匹配 → 正常 (评分=100)
IF 车牌模糊匹配+品牌匹配 → 正常 (评分=90)
```

## 三、技术实现

### 3.1 深度学习模型

| 模型 | 架构 | 输入 | 输出 | 准确率 |
|------|------|------|------|--------|
| 车辆检测 | YOLOv5s | 640×640 | bbox+类别 | 95%检测率 |
| 颜色分类 | ResNet18 | 224×224 | 8类颜色 | 94.3% |
| 车型分类 | ResNet18 | 224×224 | 4类车型 | 91.2% |
| 品牌分类 | ResNet18 | 224×224 | 47类品牌 | 训练中 |

### 3.2 车牌OCR

使用EasyOCR双引擎（ch_sim + en），配合HSV颜色过滤定位车牌区域：
- 蓝色车牌：HSV(90-140, 50-255, 50-255)
- 绿色车牌：HSV(35-85, 40-255, 40-255)
- 黄色车牌：HSV(15-40, 40-255, 40-255)
- 多策略预处理：灰度 → CLAHE增强 → OTSU二值化

### 3.3 比对算法

多因素评分系统（0-100分）：
- 车牌精确匹配：40分
- 品牌完全匹配：60分
- 品牌别名匹配：48-54分
- 模糊匹配扣10%

### 3.4 Web前后端

- 前端：Bootstrap 5 + Jinja2模板引擎
- 后端：Flask 3.1.3，RESTful API
- 功能：拖拽上传、图片预览、实时系统状态、评分条、特征标签

### 4.3 比对场景测试

| 场景 | 判定 | 评分 |
|------|------|------|
| 车牌+品牌完全匹配 | ✅ 正常 | 100 |
| 车牌匹配+品牌不符 | ⚠️ 套牌嫌疑 | 40 |

### 4.4 已知问题

1. **OCR全景图识别率低(25%)**：卡口全景图中车牌太小，需增加车牌检测步骤
2. **品牌准确率待提升**：47类品牌训练数据有限，域偏移明显
3. **颜色域偏移**：未裁剪全景图分类不准，YOLOv5裁剪后显著改善

## 六、总结与展望

### 6.1 项目成果

- 完成了一个端到端的套牌车稽查系统MVP
- 实现了车辆检测→特征提取→数据库比对→Web展示的完整管线
- 车辆检测率达95%，颜色分类准确率94.3%
- 系统支持拖拽上传、实时分析、可视化结果展示

### 6.2 改进方向

1. **OCR增强**：集成专用车牌检测模型（HyperLPR/PaddleOCR）
2. **模型微调**：使用阶段三3700张XML标注数据微调YOLOv5
3. **品牌提升**：增加训练数据量和训练轮数
4. **性能优化**：模型量化、批处理推理
5. **部署**：Docker容器化部署
